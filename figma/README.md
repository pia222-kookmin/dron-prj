# OTTOMOBI Figma UI 디자인 연동 가이드

이 폴더는 현재 Next.js로 구현된 OTTOMOBI 홈페이지의 HTML과 CSS를 정적으로 추출하여 저장한 폴더입니다. 이 파일들을 Figma로 가져가 디자인 시안을 편리하게 작업하고 수정할 수 있습니다.

## 1. Figma로 가져가기 (HTML to Design)

Figma에서 HTML을 직접 디자인 요소(레이어 및 프레임)로 변환해 주는 플러그인을 사용하여 이 파일을 가져올 수 있습니다.

1. Figma를 실행하고 디자인 파일을 엽니다.
2. 플러그인 메뉴에서 **"html.to.design"** (또는 **"Builder.io"**) 플러그인을 찾아서 실행합니다.
3. 플러그인 설정 중 **"Import from local files"**(로컬 파일 임포트)를 선택합니다.
4. 이 폴더에 있는 `index.html`과 `styles.css` 파일을 플러그인 창에 드래그 앤 드롭합니다.
5. 변환 버튼을 누르면 홈페이지의 모든 구성 요소가 피그마 프레임과 벡터 레이어로 깨짐 없이 임포트됩니다.

---

## 2. 디자인 수정 후 코드로 역반영하는 방법

피그마에서 레이아웃, 크기, 폰트, 정렬 등의 디자인 요소를 수정하신 후에 해당 내용을 다시 실제 웹사이트에 반영하고 싶으실 경우 아래 방안을 참고해 주세요:

### 방법 A: 수정된 Figma 시안을 개발자(Antigravity AI)에게 전달
- 피그마에서 변경한 디자인(예: "히어로 영역의 간격을 좁히고 제품 카드의 그림자를 더 짙게 해줘", "버튼 색상을 조금 더 남색으로 바꿨어" 등)을 피그마 링크 또는 캡처 화면과 함께 채팅창으로 말씀해 주시면, 제가 자동으로 코드 파일에 안전하게 역반영해 드립니다.

### 방법 B: Figma에서 HTML/CSS 코드로 추출하여 덮어쓰기
- Figma 플러그인 중 **"Figma to Code"** 또는 **"Figma to HTML"** 기능을 이용하여 수정된 요소의 HTML/CSS 코드를 추출합니다.
- 추출한 HTML 조각과 CSS 코드를 이 폴더(`figma/index.html`, `figma/styles.css`)에 저장하신 뒤, 저에게 **"피그마 폴더의 변경사항을 React 컴포넌트에 반영해줘"**라고 지시해 주시면 컴포넌트 구조가 깨지지 않도록 정밀 병합(Merge)해 드립니다.

---

## 3. 폴더 구조
- `index.html`: 추출된 메인 홈페이지 마크업 (Figma 연동용으로 스크립트가 정제됨)
- `styles.css`: 홈페이지 전체에 적용된 통합 스타일시트
- `images/`: 홈페이지 로고 및 기본 이미지 자산
- `uploads/`: 관리자 페이지에서 업로드한 장비 및 제품 사진 자산
